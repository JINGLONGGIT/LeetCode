### 题目分析
该题的主要陷阱是要考虑进位导致的数组大小发生变化的问题：比如`999 + 1 == 1000`

测试用例：
```
输入：[1, 2, 3]
输出：[1, 2, 4]

输入：[4, 3, 2, 1]
输出：[4, 3, 2, 2]

输入：[1, 2, 9, 9]
输出：[1, 3, 0, 0]

输入：[9, 9, 9]
输出：[1, 0, 0, 0]
```

解法一：
暴力解法，从数组的最后一位往前遍历。如果需要进位，则依次进位，如果第一位需要进位，则将数组大小增1

解法二：从第一位往后遍历，判断数组元素是否等于9。
* 1）如果有一位不等于9，则数组大小不变，只需要依次进位即可；
* 2）如果所有数组元素都等于9，则将数组大小增1，然后数组的第一个元素为1，其他全为0
